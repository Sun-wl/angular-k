新增订单

接口简介

新增配送单接口

接口调用URL地址：/api/order/add。

请求参数说明

名称 类型 必传 描述
shop_no String 是 门店编码
origin_id String 是 第三方订单ID
order_sn String 否 订单流水号
cityCode String 是 订单所在城市的code（查看各城市对应的code值)
rateTime Integer 是 时效(0:为不限 其他数值为X分钟内)
serviceMoney Integer 是 配送费(单位: 分)
from_name String 是 发货人姓名
fromMobile String 是 发货人联系方式
fromAddress String 是 发货人地址
fromAddressDetails String 否 发货人详细地址/门牌号
fromLat Double 是 发货人地址维度（高德坐标系）
fromLng Double 是 发货人地址经度（高德坐标系）
toName String 是 收货人姓名
toMobile String 是 收货人联系方式
toAddress String 是 收货人地址
toAddressDetails String 否 收货人详细地址/门牌号
toLat Double 是 收货人地址维度（高德坐标系）
toLng Double 是 收货人地址经度（高德坐标系）
tipMoney Integer 否 小费金额 (单位 : 分)
createTime Long 是 订单创建时间（时间戳,以秒计算时间，即unix-timestamp）
remark String 否 订单备注
goodsType Integer 是 订单商品类型：1、餐饮 2、包裹 3、揽件 4、鲜花 6、其他 9、生鲜
goodsList [] 否 商品对象的数组 name 为商品名称 num 为商品数量
weight Double 否 物品重量

返回结果

返回JSON格式，以下为参数说明：

参数名称 类型 描述
statusCode Integer 响应状态，成功为"200"，失败为"300"
message String 如果错误则显示错误原因
data Object 响应结果，JSON对象，详见具体的接口描述

data业务参数：

参数名称 类型 描述
orderNo String 运单号
正确的响应结果（JSON格式） 
            {"statusCode":200,"data":{"orderNo":"3581510595300014"},"msg":"成功"}
