订单回调

接口简介

每次订单状态发生变化时，会对添加的APP中的callback的URL进行回调。

1. 参数以http方式传递。若回调服务器响应失败（响应非200），会每隔1分钟重试一次，最多重试6次。

2. 每次订单状态变化都会发生回调，如果出现订单状态回调顺序不一致，请根据回调参数中的时间戳进行判断。

3. 在线上环境中，订单状态的变化会主动触发订单的回调；而在测试环境中，需要模拟订单的各个状态来触发回调。

4. 测试环境，模拟订单各状态的接口如下：

  • 模拟骑手接单
  • 模拟完成取货
  • 模拟完成订单
  • 模拟取消订单

参数说明

名称 类型 是否必传 描述
order_id String 是 第三方订单编号
status Integer 是 订单状态 (1:待接单 2:已接单等待取货 3:已取货正在配送 5:已取消 7:已完成 8:订单异常)
cancelReason String 是 取消原因
cancelFrom String 是 订单取消原因来源(1:配送员取消；2:商家主动取消；3:系统或客服取消；0:默认值)
updateTime Long 是 更新时间
runnerId Long 否 配送员ID (接单后会传)
runnerName String 否 配送员姓名 (接单后会传)
runnerMobile String 否 配送员联系方式 (接单后会传)

状态说明

已取消：包括配送员取消、商户取消、系统取消（待接单、待取货、配送中的订单）
