订单状态查询

接口简介

查询订单的状态以及骑手的相关信息。

接口调用请求说明，URL地址：/api/order/status/query。

请求参数说明

名称 类型 是否必传 描述
order_id String 是 第三方订单编号

返回结果

返回JSON格式，以下为参数说明：

参数名称 类型 描述
statusCode Integer 响应状态，成功为"200"，失败为"300"
message String 如果错误则显示错误原因
data Object 响应结果，JSON对象，详见具体的接口描述

data业务参数：

参数名称 类型 描述
origin_id String 第三方订单编号
status Integer 订单状态编码
statusText String 订单状态
runnerName String 配送员姓名
runnerMobile String 配送员联系方式
runnerLat Double 配送员当前位置纬度（高德坐标系）
runnerLon Double 配送员当前位置经度（高德坐标系）
serviceMoney Integer 配送费(单位: 分)
tipMoney Double 小费
distance Double 距离
createTime Long 创建时间
receiveTime Long 接单时间
takeTime Long 取货时间
finishTime Long 完成时间
cancelTime Long 取消时间

只有当订单被接单后才会有骑手信息，并且在待取货和配送中可以查询骑手实时的坐标信息。
